Common questions

Why aren't the nav bars using the Uno NavigationBar component? ▾
The designs need custom layouts with status indicators, action icons, and badges that the standard NavigationBar doesn't support. The custom nav bar is a simple 3-column Grid — left icon, center title, right actions. It's 10 lines of XAML.
How do I know which Figma component maps to which XAML control? ▾
Every component card above shows the exact XAML tag next to the component name (the blue tag on the right). The Figma variant values map directly to XAML style properties. For example, Button Style="Filled" in Figma → Style="{StaticResource FilledButtonStyle}" in XAML.
What about the 70% that isn't Uno components? ▾
It's standard XAML layout: StackPanel, Grid, TextBlock, Border. No custom renderers, no platform-specific code. The "Custom compositions" section above has copy-paste XAML for every pattern. The most complex one (nav bar) is 10 lines.
Why 0px corners on everything? ▾
It's the Avontus brand identity — sharp geometric aesthetic. The Uno Material components ship with rounded corners by default. We customized the design system to override all radii to 0px. This is handled in the design token layer.
How should offline sync actually work? ▾
The designs show the UI pattern: a red "No connection" banner with queued change count. The actual sync logic is backend — queue changes in local storage, sync when connectivity returns. The UI just needs to display the connection state and queue count.
What's the difference between MenuItem and ListItem? ▾
MenuItem = navigation row with a chevron arrow. Use for "Who Am I", "Sample Form", etc. ListItem = data row where Trailing=True shows a ToggleSwitch. Do NOT use ListItem for navigation — you'll get a toggle instead of a chevron.
Do I need to implement the Switzer font, or does Uno handle it? ▾
You need to bundle Switzer as a custom font. Uno Material ships with Roboto. We replaced it at the design system level. Include Switzer Regular (400) and Medium (500) in your app resources and set it as the default font family.
What about dark mode? ▾
Not in scope for the current POC. All designs are light mode only. If we add dark mode later, the color tokens are structured to support it — just provide alternate values per theme.
