Gray floor: no text lighter than #777777 Field workers use this app in direct sunlight. Anything lighter than #777 becomes invisible outdoors. This is enforced in the design system.

Typography

Switzer from Fontshare. Two weights: Regular (400) for body, Medium (500) for headings. Tight negative letter-spacing.

Shape — 0px corners everywhere

The defining visual trait of Avontus: sharp geometry. Every element has 0px border-radius — buttons, cards, inputs, modals.

Exceptions (only two) Error/success banners use 12px radius. The avatar circle uses full radius. Everything else: 0px.

Field-ready UX — the 6 rules

Users are yard workers, delivery drivers, and site supervisors. Outdoors, gloves on, direct sunlight, dirty screens. Every decision flows from this.

01
56px minimum touch target
Every tappable row is at least 56px tall. Gloved fingers on wet screens miss small targets.
02
14px minimum readable text
Error messages, labels, helper text — all 14px. Body text that users act on is 16px.
03
No gray lighter than #777
In direct sunlight, light grays vanish. Primary text: #111. Secondary: #555. Tertiary floor: #777.
04
Inline validation, never modals
Error text appears directly under its field. A worker at a scaffold can't navigate popups.
05
Offline-first with visible queue
Show connection status always. When offline, show queued change count: "3 changes queued — will sync when online."
06
Minimal cognitive load
Home: 3 nav items. Login: 2 fields, 2 buttons. Get in, complete the task, get out.

How validation works

Inline field error

14px Switzer Medium, #D32F2F, placed directly below the field with 6px top margin.

<!-- Error appears right under the field -->
<StackPanel>
  <TextBox PlaceholderText="Username" />
  <TextBlock Text="Username required"
             FontSize="14" FontWeight="Medium"
             Foreground="#D32F2F"
             Margin="0,6,0,0" />
</StackPanel>

Error summary banner

Sticky at top of form, shows total error count + clear instruction.

<StackPanel Background="#FDECEA" Padding="24,14">
  <TextBlock Text="7 validation errors"
             FontSize="16" FontWeight="SemiBold"
             Foreground="#D32F2F" />
  <TextBlock Text="Fix the highlighted fields before saving."
             FontSize="14" Foreground="#D32F2F" />
</StackPanel>

How offline states work

Status dot in nav bar

8px circle + 14px label. Green #2E7D32 = online, Red #D32F2F = offline.

<StackPanel Orientation="Horizontal" Spacing="6">
  <Ellipse Width="8" Height="8" Fill="#2E7D32" />
  <TextBlock Text="Online" FontSize="14"
             FontWeight="Medium" Foreground="#2E7D32" />
</StackPanel>

Offline banner

Full-width red banner below nav bar. Shows "No connection" + queued count.

<StackPanel Background="#FDECEA" Padding="24,14">
  <TextBlock Text="No connection"
             FontSize="16" FontWeight="SemiBold"
             Foreground="#D32F2F" />
  <TextBlock Text="3 changes queued — will sync when online"
             FontSize="14" Foreground="#D32F2F" />
</StackPanel>

What's an Uno component, what's custom layout

About 30% of what you see on screen are Uno component instances. The other 70% is standard XAML layout — StackPanels, Grids, TextBlocks. Both are straightforward to implement.

Uno components (interactive controls — you get behavior for free)

Button TextBox PasswordBox ToggleSwitch MenuItem ListItem Divider Chip Card CheckBox DatePicker

Custom layout (just StackPanel + TextBlock — no special behavior needed)

Nav bars Error banners Success banners Offline banners Status dots Avatars Profile rows Footers Server URL prefix "or" divider

Why this split is normal Uno components handle the hard parts: text input, focus, keyboard, toggle animation, accessibility, cross-platform rendering. The custom layout is just positioning and text — 3-5 lines of XAML each.

Custom compositions — copy-paste XAML

These elements appear across multiple screens. They're not Uno components — they're standard layout code.

Nav bar (used on every screen)

<!-- 56px custom nav bar with back/title/action -->
<Grid Height="56" Padding="16,12" Background="White">
  <Grid.ColumnDefinitions>
    <ColumnDefinition Width="Auto" />
    <ColumnDefinition Width="*" />
    <ColumnDefinition Width="Auto" />
  </Grid.ColumnDefinitions>

  <!-- Left: icon (close, back, or menu) -->
  <Button Grid.Column="0" Style="{StaticResource IconButtonStyle}" />

  <!-- Center: title -->
  <TextBlock Grid.Column="1" Text="Connection settings"
             FontSize="18" FontWeight="SemiBold"
             HorizontalAlignment="Center"
             VerticalAlignment="Center" />

  <!-- Right: status or actions -->
  <StackPanel Grid.Column="2" Orientation="Horizontal" />
</Grid>

Server URL with "https://" prefix

<Grid>
  <Grid.ColumnDefinitions>
    <ColumnDefinition Width="Auto" />
    <ColumnDefinition Width="*" />
  </Grid.ColumnDefinitions>

  <Border Grid.Column="0"
          Background="#F5F5F5"
          BorderBrush="#D4D4D4" BorderThickness="1"
          Padding="14,14,10,14">
    <TextBlock Text="https://"
               Foreground="#777777"
               VerticalAlignment="Center" />
  </Border>

  <TextBox Grid.Column="1"
           Text="myinstance.mycompany.com" />
</Grid>
